# Security Evaluation of Perceptual Image Hashing

### Attack Efficiency Analysis on NeuralHash vs pHash

---

## Overview

This project extends the original *Learning-to-Break-Deep-Perceptual-Hashing* repository into a structured **attack efficiency evaluation framework**.

We do **not** propose a new hash design.

Instead, we evaluate:

* How Hamming distance changes under attack
* How quickly an adversary can manipulate hashes
* How attack efficiency differs between:

  * NeuralHash (96-bit deep perceptual hash)
  * pHash (64-bit classical perceptual hash)

Primary focus:

> Distance vs Steps
> Distance vs Time
> Distortion vs Decision Boundary Crossing

## Research Objective

Quantify attack efficiency by measuring:

* Change in Hamming distance vs attack step
* Change in Hamming distance vs elapsed time
* Steps required to cross decision threshold
* Distortion required to manipulate decision boundary

---

## Threat Models

### Evasion (False Negative Scenario)

Given image x and its reference hash h_ref:

Goal:
Increase Hamming distance so that

dist(hash(x'), h_ref) > T

Measure:

* Distance vs steps
* Distance vs time
* Steps-to-cross-threshold
* Distortion at crossing

---

### Targeted Collision (False Positive Scenario)

Given image x and target hash h_target:

Goal:
Reduce Hamming distance so that

dist(hash(x'), h_target) ≤ T

Measure:

* Distance decay curve
* Convergence speed
* Success rate within budget

---

## Metrics

For both NeuralHash and pHash:

* Raw Hamming distance
* Normalized Hamming distance (d / bit_length)
* L2 distortion
* L∞ distortion
* SSIM
* Success flag

Derived metrics:

* Δd per step
* Δd per second
* Area Under Distance Curve (AUC)
* Median steps-to-success
* 75th / 90th percentile time-to-success

## Logging Format

All experiments will generate long-format CSV logs.

Columns:

```
image_id
hash_method
attack_type
step
elapsed_ms
dist_raw
dist_norm
l2
linf
ssim
success
```

Each intermediate step is logged.

This enables full attack-curve reconstruction.

## Experimental Protocol

### Dataset (Initial Phase)

* 50–200 natural images
* Fixed resolution
* No preprocessing bias

### Attack Budgets

* Max steps: 500–1000
* Max time per image: fixed wall-clock limit
* Max distortion constraint

Each run logs every step.

## Expected Contribution

Rather than asking:

“Can perceptual hashing be broken?”

We ask:

* How fast can it be broken?
* At what distortion cost?
* How does decision boundary stability differ across hash families?

This reframes perceptual hashing security from binary success/failure to continuous attack-efficiency analysis.
